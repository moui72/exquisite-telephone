# Plan — Lobby displays count active players

## Goal

Make the Lobby's live below-minimum warning and live laps-per-book
default compute over the active (non-kicked) roster, so what a host reads
before starting matches what the server does and what `datamodel.md`
already states.

## Scope

**In scope.** The two raw-roster reads in `client/src/lib/views/
Lobby.svelte`: `belowMinimumPlayers` (`:40`) and the live `lapsPerBook`
default (`:84`). Both switch to `activePlayers(state.room).length`. Plus
`ui.md`'s Lobby View wording for these two displays, which currently
describes them without the active-count qualification — bringing it in
line with `datamodel.md`'s Minimum-player and Laps-per-book rules
(corrected in `plan-4663`).

**Not in scope.** The server-side enforcement and `datamodel.md` — both
already count active players (`plan-4663`). Any other Lobby control. This
plan closes the *display* half of the same seam; the logic half already
landed.

## Technical Approach

`activePlayers` is already imported in `Lobby.svelte` and used for the
roster render (`:210`), and `WritingDrawing.svelte:54` already computes
its equivalents over it — so this is making the Lobby internally
consistent, not adding a dependency. The two reactive statements change
from `state.room?.players.length ?? 0` to
`activePlayers(state.room).length` (guarding the `null` room the same way
the current `?? 0` does).

The behavior this corrects is exactly the mismatch the feedback names: a
host who kicks a lobby down can currently see "enough to start" and a
1-lap default, then have the server reject the start or resolve 2 laps.
After this, the display agrees with the server before the host acts.

`ui.md`'s Lobby View has two sentences to align: the player-count
guidance ("below 3 players … checkbox appears") and the laps live-default
("recalculates as players join or leave"). Both gain the active-count
qualification, matching the `datamodel.md` edits `plan-4663` already made
— a doc catching up to a now-consistent code+datamodel pair, not new
drift.
