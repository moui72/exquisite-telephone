# Plan: Fix two salon-gallery-ui-redesign follow-up issues

## Goal

Fix the two open feedback items from `salon-gallery-ui-redesign`'s
implementation: the host seeing the "game can't continue" notice twice,
and Lobby's Error/Empty states showing raw server error codes instead of
docent-voice copy.

## Scope

**In scope:**
- F001: suppress the page-body "game can't continue" notice
  (`WritingDrawing.svelte`) for the host specifically, since the
  Moderation Panel already shows it to them; non-host players (who have
  no panel) keep seeing it in the page body.
- F002: add an error-code-to-docent-voice-copy mapping in
  `Lobby.svelte`, covering every error code reachable from a
  Lobby-triggered action (create room, join room, start game, rejoin),
  replacing the current verbatim `{state.error}` render.
- Small clarifying edits to `ui.md`'s Moderation Panel and States
  sections recording both decisions, so the "single visible copy of the
  notice" and "codes are translated, not shown raw" rules are documented
  design, not just code behavior.

**Out of scope:**
- Any other error code not reachable from Lobby (e.g. in-round codes
  like `not-your-turn`, `round-not-open`, `book-not-found`) — those
  surface elsewhere or not at all in the current UI and aren't part of
  either feedback item.
- Any change to when/why `Room.nonContinuable` is set, or to the
  Moderation Panel's own notice — only the *duplicate* body copy is
  touched.

## Technical Approach

**F001** — `WritingDrawing.svelte`'s existing `{#if state.room.nonContinuable}`
alert block gets one additional condition: only render when the current
player is *not* the host (`state.player?.id !== state.room.hostPlayerId`).
The host relies solely on `ModerationPanel`'s own copy of the notice,
which is already host-gated (`{#if isHost && room}`). No change to
`ModerationPanel.svelte` itself.

**F002** — a small lookup object (e.g. `ERROR_COPY: Record<string,
string>` local to `Lobby.svelte`, or a new tiny module if that reads
cleaner) maps every error code the server can emit that's reachable from
a Lobby-triggered action — `room-not-found`, `not-host`,
`too-few-players`, `room-not-in-lobby`, `invalid-token`, `game-ended` —
to a docent-voice line, with a sensible generic fallback line for any
future/unmapped code (so an unmapped code degrades gracefully instead of
leaking a raw string). `Lobby.svelte` renders the mapped copy instead of
`{state.error}` directly.

Both fixes are presentation-layer only — no server, socket, or data
model changes.
